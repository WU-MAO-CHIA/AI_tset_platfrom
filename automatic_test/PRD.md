# Automatic test
## 專案目的
建立一個自動化測試平台，讓user可以在上面進行測案管理、串接資料庫撈取測資、執行網頁測試，並產生測試報告

## 測案管理
測案管理分為兩大部分，第一是建立一個測試案例資料庫，第二是為測試清單

### 測試案例資料庫
儲存過去建立的測試案例，提供不同專案和測試人員挑選做使用，每次修改時需要更新版本號碼，並需要有個建立與測試案例的畫面

#### 建立測試案例
畫面欄位要有
- 編號
- 測試案例名稱
- 測試案例說明
- 測試資料(允許user新增數量)
- 前置步驟
- 主要步驟
- 建立/修改人員
- 系統別
- 標籤
- 版本號

#### 測試案例清單
條列式顯示測試案例，要有篩選功能(by 系統別、名稱等)且支持多條件，點開後可以進入更詳細的畫面

#### 建立測試清單
user可以從測試案例清單中挑選相關測試案例匯入，畫面中要有地方可以填寫測試清單名稱，建立人員與紀錄每次執行的相關紀錄

# Automatic test
## 專案目的
建立一個自動化測試平台，讓user可以在上面進行測案管理、串接資料庫撈取測資、執行網頁測試，並產生測試報告

## 測案管理
測案管理分為兩大部分，第一是建立一個測試案例資料庫，第二是為測試清單

### 測試案例資料庫
儲存過去建立的測試案例，提供不同專案和測試人員挑選做使用，每次修改時需要更新版本號碼，提供新增、刪除、修改和查詢的功能，並需要有個建立測試案例的畫面

#### 建立測試案例
畫面欄位要有下列欄位，並串接各大主流LLM模型，透過AI模型理解和補齊測試步驟，支持user透過上傳圖片、影片、網址等方式，提供多樣化的資訊，建立完測試案例後，立刻使用測試引擎進行轉譯，快速執行一次測試案例供user確認是否跟user想像相同
- 編號
- 測試案例名稱
- 測試案例說明
- 測試資料(允許user新增數量)
- 前置步驟
- 主要步驟
- 建立/修改人員
- 系統別
- 標籤
- 版本號


#### 測試案例清單
條列式顯示測試案例，要有篩選功能(by 系統別、名稱等)且支持多條件，點開後可以進入更詳細的畫面顯示測試案例，支持上傳excel、CSV或文字檔等方式提供測試所需要之相關測試參數

#### 建立測試清單
user可以從測試案例清單中挑選相關測試案例匯入，畫面中要有地方可以填寫測試清單名稱，建立人員與紀錄每次執行的相關紀錄

## 測試引擎
使用Robot-Framework為測試的主要框架，支持任何大小的網頁測試，且需要支持平行執行的功能

### 測試代碼
將user建立的測試案例透過AI翻譯成 Robot-Framework 的程式碼


## 測試結果
以Robot-Framework提供的xml為來源基礎，將結果透過網頁顯示，讓user可以從測試案例清單中快速找到每次執行的結果，如果測試中有相關影片或圖片也一併顯示在網頁上
